# English Quest Roadmap

## 当前判断

English Quest 的核心学习闭环已经能跑通。v1 接下来最有价值的工作不是继续堆功能，而是优化用户每天打开 App 时的主路径：从首页进入学习、理解赛道、查看自己的学习成果。

## v1 优先级

### P0：赛道体验重做

目标：让赛道页从“管理列表”变成“学习路线选择”。

- 重做 `TracksPage` 的赛道卡片视觉层级。
- 强化每条赛道的主题感：旅行、追剧、应试要明显不同。
- 主操作聚焦到 `继续学习`，`查看地图`作为次级操作。
- 用 `已完成关卡 / 总关卡`、当前章节、星级等信息替代弱百分比表达。
- 后续把 `TrackMapPage` 从 ReactFlow 工具画布改成移动端纵向关卡路径。

### P1：首页任务中心

目标：用户打开首页后能立刻知道今天该做什么。

- 顶部保留问候、等级、XP、连续打卡。
- 主卡片只突出一个推荐任务：继续当前赛道的下一关。
- 复习提醒作为高优先级任务出现。
- 随机挑战和选择赛道作为辅助入口。
- 本周统计保持简洁，不再堆过多卡片。

### P1：我的页面学习档案

目标：从“设置页”升级为“学习档案 + 成就反馈”。

- 强化用户头像、昵称、等级、XP、连续打卡。
- 本周统计展示已学词、完成关卡、平均星级、错题待修复。
- 设置、关于等低频操作下沉。
- 未实现的编辑头像/昵称按钮要么补齐，要么暂时隐藏。

### P2：PWA 和移动端 polish

- 补齐 PWA icon。
- 做真机测试清单。
- 修复移动端溢出、底部导航遮挡、Safari 行为差异。

## v2 候选方向

v2 再考虑需要真实后端或外部接口的能力：

- 接入真实 AI API，替换 `src/ai/mock.ts` 中的 mock 示例生成。
- 自定义词包。
- 云端同步学习进度。
- 更复杂的听力填空和口语反馈。
- Capacitor / Electron 等壳应用。

## 不建议现在做

- 不建议现在大规模接 AI API，除非先明确后端代理和失败兜底。
- 不建议同时重构首页、赛道、我的、学习页，容易造成视觉和功能都半成品。
- 不建议继续把所有阶段记录追加到 `CLAUDE.md`，后续应按主题拆到 `docs/`。
