# English Quest Roadmap

*更新：2026-04-29*

---

## 产品方向确认

English Quest 是一款**场景驱动的综合英语学习 PWA**。当前阶段以词汇为基础，后续扩展口语、语法、听力。

核心设计原则：
- **语境例句优先**：用户通过真实场景理解单词，而非死记硬背
- **专注单一主题**：用户选择一个感兴趣的场景主题，学透后再切换
- **全功能免费**：不设金币墙、付费锁

## v1 当前优先级

### P0：Travel L1 词库补全 🔴 进行中

目标：将 travel 轨道 L1 的词库从占位符替换成真实高质量内容。

- 使用多 Agent 流水线（见 `docs/CONTENT_AGENT_WORKFLOW.md`）
- 每个词需要：真实例句、准确中文释义、相关词
- 完成后 drama L1 跟进

### P1：主动回忆按钮 🟠 待开发

目标：在词汇弹窗（WordCard）底部将"返回学习"改为两个选项。

```
[😅 还不熟，再复习]    [✅ 记住了，继续]
```

用户看完词义后主动判断，显著提升记忆留存率。实现成本低，优先级高。

### P2：主题页连续打卡可视化 🟡 待优化

当前连续打卡显示为"N 天连续"文字，可升级为：
- 日历热力图（当前在 MePage，可考虑在主题页增加 mini 版）
- 或周视图（7 格，绿色=打卡，灰色=未打卡）

### P3：PWA 和移动端 polish 🟡 待处理

- 补齐 PWA icon
- 真机测试清单（iOS Safari、Android Chrome）
- 修复底部导航遮挡内容

## v2 候选方向

v2 再考虑需要真实后端或外部接口的能力：

- **更多主题**：职场面试、海外生活、日常生活等
- **口语练习**：Web Speech API 录音 + AI 反馈
- **语法模块**：语法点场景化教学
- **云端同步**：学习进度跨设备同步
- **社交排行榜**：朋友间比较学习进度

---

## 不建议现在做

- 不建议恢复应试派为独立主题，考试词汇通过等级难度自然覆盖
- 不建议同时重构多个页面，容易造成视觉和功能都半成品
- 不建议大规模接 AI API，除非先明确后端代理和失败兜底
- 不建议继续把所有阶段记录追加到 `CLAUDE.md`，后续应按主题拆到 `docs/`
