# English Quest 文档中心

这个目录用于管理项目的长期文档。根目录只保留少量入口文档，详细方案、路线图、设计说明和历史记录逐步放到 `docs/` 下。

## 文档地图

| 文档 | 用途 | 维护频率 |
| --- | --- | --- |
| `../README.md` | 项目入口：介绍、启动方式、当前状态、关键链接 | 每次阶段变化后更新 |
| `../CLAUDE.md` | 开发上下文和历史实施记录，给 AI/开发协作使用 | 完成较大阶段后补充 |
| `../UI_OPTIMIZATION_PLAN.md` | 首页、赛道、我的页面的 UI/UX 优化计划 | 执行 UI 任务前后更新 |
| `../ILLUSTRATION_GUIDE.md` | 插画资源下载、命名和接入指南 | 插画规范变化时更新 |
| `ROADMAP.md` | v1/v2 优先级和近期执行顺序 | 每轮规划后更新 |
| `DOCS_GUIDE.md` | Markdown 文档维护规则 | 文档结构变化时更新 |

## 当前重点

v1 当前不建议优先扩展大功能，建议聚焦三个体验入口：

1. `首页`：改成今日任务中心，强化“下一步该做什么”。
2. `赛道`：重做视觉层级和路径感，这是当前最影响观感的模块。
3. `我的`：从设置菜单升级为学习档案，强化成就感和学习反馈。

## 推荐维护方式

- `README.md` 只写外部读者最需要的信息，不堆实施细节。
- `CLAUDE.md` 保留历史决策和阶段记录，但后续避免继续无限追加。
- 具体计划写到独立文档，例如 UI 优化写 `UI_OPTIMIZATION_PLAN.md`，路线图写 `docs/ROADMAP.md`。
- 完成一个阶段后，把“计划状态”改成“已完成/部分完成/废弃”，避免旧计划误导后续开发。
